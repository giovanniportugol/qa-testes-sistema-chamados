# Plano de Testes - Sistema de Chamados

## 1. Objetivo

Este plano de testes tem como objetivo definir a estratégia de testes para um sistema fictício de chamados, simulando um ambiente utilizado para registro, acompanhamento e resolução de solicitações de suporte.

O projeto faz parte do meu portfólio de estudos em QA/Testes de Software.

## 2. Escopo dos Testes

Serão testadas as principais funcionalidades do sistema:

- Login de usuário
- Cadastro de chamados
- Consulta de chamados
- Edição de chamados
- Alteração de status
- Validação de campos obrigatórios
- Mensagens de erro e sucesso

## 3. Tipos de Testes

- Testes funcionais
- Testes manuais
- Testes positivos
- Testes negativos
- Testes de validação de campos
- Testes de usabilidade básica

## 4. Ambiente de Teste

- Sistema: Sistema fictício de chamados
- Navegador: Google Chrome
- Sistema operacional: Windows
- Tipo de teste: Manual

## 5. Critérios de Entrada

Para iniciar os testes, é necessário que:

- As telas principais estejam disponíveis
- As funcionalidades básicas estejam implementadas
- O usuário tenha acesso ao sistema
- Os requisitos estejam definidos de forma básica

## 6. Critérios de Saída

Os testes serão considerados concluídos quando:

- Os principais cenários forem executados
- Os resultados forem registrados
- As falhas forem documentadas
- As evidências forem armazenadas
- Os casos críticos forem validados

## 7. Riscos

- Requisitos incompletos ou pouco detalhados
- Falta de ambiente real para execução
- Dados de teste limitados
- Alterações nas funcionalidades durante os testes
